# Habimori — User Flow (MVP)

## Общая идея

Habimori — это приложение, где пользователь:

* создаёт **цели** (на время, количество или факт выполнения),
* логирует действия максимально быстро,
* всегда начинает с **главного экрана**,
* может создавать **контексты и теги на лету**, без предварительных настроек.

---

## Flow 0. Первый вход в приложение

**Цель:** пользователь понял, что делать дальше.

1. Пользователь заходит в приложение.
2. Авторизуется через Google.
3. Попадает на главный экран.

Если целей ещё нет:

* видит пустое состояние;
* видит большую кнопку **«Добавить цель»**.

---

## Flow 1. Создание цели — счётчик (пример: отжимания)

**Сценарий:** «Хочу сделать 1000 отжиманий за месяц».

1. На главном экране нажать **«Добавить цель»**.
2. Открывается экран создания цели.
3. Пользователь заполняет:

   * Название: `1000 отжиманий`
   * Тип цели: `Счётчик`
   * Период: `Месяц`
   * Тип цели: `Позитивная`
   * Целевое значение: `1000`
   * Дата начала: сегодня
   * Дата окончания: через месяц
4. Поле **Контекст**:

   * пользователь вводит `Спорт`;
   * контекста ещё нет → он создаётся автоматически.
5. Поле **Теги**:

   * пользователь добавляет `отжимания`, `фитнес`.
6. Нажимает **Сохранить**.

Результат:

* цель появляется на главном экране;
* она активна и готова к использованию.

---

## Flow 2. Создание цели — таймер (пример: английский)

**Сценарий:** «Учить английский 5 часов в неделю».

1. На главном экране нажать **«Добавить цель»**.
2. Заполнить:

   * Название: `Английский`
   * Тип цели: `Время`
   * Период: `Неделя`
   * Тип цели: `Позитивная`
   * Целевое значение: `5 часов`
   * Дата начала: сегодня
   * Дата окончания: конец года
3. Контекст:

   * ввести `Обучение` → создаётся новый контекст.
4. Теги:

   * `английский`
5. Сохранить.

Результат:

* на главном экране теперь две цели;
* каждая отображает свой прогресс.

---

## Flow 3. Главный экран — рабочее место пользователя

**Главный экран — это не “дашборд”, а пульт управления.**

Пользователь видит:

* список активных целей;
* прогресс по текущему периоду;
* быстрые действия прямо на карточках целей.

### Для счётчика:

* кнопка `+1`;
* кнопка «ввести число».

### Для таймера:

* кнопка `Play / Stop`.

---

## Flow 4. Логирование счётчика (пример: отжимания)

**Сценарий:** «Сделал 30 отжиманий».

1. Пользователь на главном экране.
2. На карточке цели `1000 отжиманий`:

   * нажимает `+1` несколько раз
     **или**
   * нажимает “ввести число” → вводит `30`.
3. Действие сохраняется.

Результат:

* прогресс цели обновляется сразу;
* пользователь ничего не настраивал дополнительно.

---

## Flow 5. Логирование времени через цель (пример: английский)

**Сценарий:** «Начал учить английский».

1. На главном экране нажать `Play` на цели `Английский`.
2. Таймер запускается сразу:

   * цель уже известна;
   * контекст уже известен.
3. Пользователь занимается.
4. Нажимает `Stop`.

Результат:

* сессия времени записана;
* прогресс недельной цели обновлён.

---

## Flow 6. Глобальный быстрый таймер (без цели)

**Сценарий:** «Работаю, но это не цель».

1. Пользователь нажимает глобальную кнопку `Play`.
2. Приложение просит:

   * выбрать или создать **контекст** (обязательно);
   * выбрать теги (опционально);
   * цель — можно не выбирать.
3. Таймер запускается.
4. Пользователь нажимает `Stop`.

Результат:

* время сохранено;
* данные можно потом использовать в статистике.

---

## Flow 7. Просмотр цели и ручное добавление данных

**Сценарий:** «Хочу посмотреть детали и добавить сессию вручную».

1. Пользователь нажимает на карточку цели.
2. Открывается экран цели.
3. Он видит:

   * описание цели;
   * прогресс;
   * список событий (сессии, инкременты).
4. Может добавить событие вручную:

   * выбрать дату;
   * указать время начала и конца (для таймера)
   * или указать количество (для счётчика).

Результат:

* данные добавлены;
* прогресс пересчитан.

---

## Flow 8. Навигация и вторичные экраны

* С главного экрана можно:

  * выйти из аккаунта;
  * перейти в “Статистику” (в MVP — заглушка).
* Отдельных экранов управления контекстами и тегами **может не быть** — всё создаётся на лету.

---

## Ключевые принципы MVP (важно зафиксировать)

* **Главный экран — центр всего.**
* **Минимум шагов для логирования.**
* **Контекст обязателен, цель — опциональна.**
* **Создание сущностей без перехода в “настройки”.**
* **Никаких обязательных статистик в MVP.**

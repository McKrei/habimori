## Экран статистики — план работ

### Решения
- Графики: все перечисленные (статусы целей, общее время, time by context, pie).
- Фильтр периода: сегментированная кнопка; поля `от/до` только для «Произвольный».
- Неделя: ISO (Пн–Вс).
- Иконка перехода в /stats: рядом с кнопкой выхода справа.

### Задачи
1. Проанализировать текущий `/stats` и сопоставить с требованиями (фильтры, графики, данные).
2. Добавить иконку перехода в `/stats` в шапку (кнопка как у выхода, inline SVG).
3. Переработать фильтр периода: объединённый контрол (Неделя/Месяц/Произвольный), дефолт — Месяц.
4. Добавить/уточнить кастомный диапазон дат (поля `от` и `до`), валидация и обновление запросов.
5. График 1: количество периодов целей по статусам (success/fail/in_progress) — 3 линии, фиксированные цвета.
6. График 2: суммарное отслеживаемое время — линия по дням периода.
7. Уточнить логику фильтрации по Context/Tags (multi) и обновить тексты/лейблы.
8. Обновить `docs/screens.md` и `docs/overview.md` при изменении поведения/состава графиков.
9. Проверить UI через Playwright после блока изменений.

## [x] Фича 0. Каркас приложения (начинаем с этого)

Цель: получить живую навигацию и пустые экраны для наращивания функционала.

Задачи:
- Роуты (пустые экраны):
  - `/` (главная)
  - `/goals/new` (создание цели)
  - `/goals/[id]` (детали цели)
  - `/stats` (заглушка)
- Layout:
  - шапка: название + Logout (пока заглушка)
  - контейнер с базовой сеткой
  - нижняя панель с глобальной кнопкой Play (пока кнопка)
- Пустые состояния:
  - на `/` — CTA "Добавить цель"

Как проверить:
1) `npm install`
2) `npm run dev`
3) Открыть `http://localhost:3000`
4) Перейти на `/goals/new`, `/goals/1`, `/stats` и убедиться, что страницы рендерятся
5) Проверить наличие шапки, контейнера и нижней панели с кнопкой Play

Проверка через MCP:
- MCP (Playwright): открыть `http://localhost:3000`
- Порт: `3000`
- Учетные данные: не требуются
- При необходимости использовать Context7 для справок по Next.js
- Статус: успешно — проверены `/`, `/goals/new`, `/goals/1`, `/stats`

Автотесты (желательно):
- E2E (Playwright): smoke-тесты маршрутов `/`, `/goals/new`, `/goals/1`, `/stats`
- Запуск: `npm run test:e2e` (после добавления Playwright)

## [ ] Фича 3. Создание цели

Цель: пользователь создает цель через форму.

Задачи:
- `/goals/new`:
  - поля цели (MVP-минимум):
    - title
    - goal_type: time | counter | check
    - period: day | week | month

    - target_op (переключатель, default gte):
      - “Хочу не меньше (позитивная цель)” => gte

    - end_date (обязателен)
  - выбор/создание контекста на лету
  - теги убрать (опционально вернуться позже)
- Сохранение в `goals` (+ создание `contexts` при необходимости)

Как проверить:
1) Создать цель на `/goals/new`
2) Убедиться, что запись появилась в таблице `goals`
3) Проверить, что контекст создается при новом имени

Проверка через MCP:
- MCP (Playwright): заполнить форму и отправить
- Адрес: `http://localhost:3000/goals/new`
- Учетные данные: авторизованный пользователь

Автотесты (желательно):

## [ ] Фича 5. Базовое логирование с главной

Цель: минимальный рабочий MVP-логинг без деталей.

Задачи:
- Для `counter`:
  - кнопка `+1` (сразу пишет событие)
  - кнопка `+N` (инлайн-ввод на карточке):
    - показ поля “Количество” + кнопки OK / Отмена
    - Enter = OK, Esc/Отмена = закрыть
    - после OK поле очищается и скрывается
    - валидация: целое число > 0
- Для `time`:
  - `Play` создает `time_entry` (ended_at = null)
  - `Stop` закрывает текущий `time_entry`
- Для `check`:
  - toggle “Сделано”
  - `On` => создаём `check_event` со `state=true`
  - `Off` => создаём `check_event` со `state=false`
  - это события, а не обновление состояния

Как проверить:
1) Открыть `/` с целями `counter` и `time`
2) Нажать `+1` и `+N` и проверить записи в `counter_events`
3) Нажать `Play` и `Stop` и проверить записи в `time_entries`
4) Переключить toggle на check-цели и проверить записи в `check_events`

Проверка через MCP:
- MCP (Playwright): нажать кнопки на карточках цели
- Адрес: `http://localhost:3000`
- Учетные данные: авторизованный пользователь

Автотесты (желательно):
- E2E: инкремент счетчика и старт/стоп таймера
- Запуск: `npm run test:e2e`

## [ ] Фича 8. Полировка (после MVP)

Задачи:
- goal_periods + архивирование:
  - статус `archived` для `goal_periods`
  - `goals.is_archived = true` при архивировании цели
  - архивные цели скрываются из UI, но периоды учитываются в статистике
- пересчет `goal_periods`:
  - при создании/изменении цели
  - при добавлении/удалении событий
  - при архивировании цели (ставим `archived`)
- архивирование цели (удаление = архив):
  - цель скрывается из главной и деталей
  - периоды сохраняются (success/fail/in_progress/archived)
- редактирование цели:
  - title
  - теги (добавление/удаление)
  - end_date не раньше today
- удаление событий по цели (time/counter/check)
- фильтры на главной (multi-select):
  - контексты
  - теги
  - статусы goal_periods
  - кнопка Reset

Как проверить:
1) Создать цель, добавить события → проверить `goal_periods` (success/fail/in_progress)
2) Архивировать цель → цель скрывается, `goal_periods.status = archived`
3) Изменить title/tags/end_date → обновление в UI и БД, end_date не раньше today
4) Удалить событие → исчезает из списка, `goal_periods` пересчитаны
5) Фильтры на главной: multi-select по контекстам/тегам/статусам + Reset

Проверка через MCP:
- MCP (Playwright): проверить фильтры/редактирование
- Адрес: `http://localhost:3000`
- Учетные данные: авторизованный пользователь

Автотесты (желательно):
- E2E: фильтры и редактирование
- Unit: расчет прогресса и статусов
- Запуск: `npm test` / `npm run test:e2e`
